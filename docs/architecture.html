RuleWhat it meansWhy

push-to-talk only空格 = 一次发声。不区分长短按,不支持打断。清理 in-flight PCM 太麻烦。

5Design decisions

TopicChoiceWhyBucket
Streaming JSON extractionSpeechExtractor 解析单个 LLM 响应。省一次 LLM 调用 + 首字延迟最优;~100 行。performance
TTS voiceFixed seed-tts-2.0-standard + Tina老师2.0 + cache。稳定性优先,开场白 100% 一致。reliability
No client splittingLLM token 直接喂 TTS。豆包推荐;更自然 + 情绪饱满。ux
Echo suppression仅 getUserMedia 三标志位。简单,无服务端 DSP。simplicity
ASR per-turn / TTS long-conn不同 socket 生命周期。TTS 握手 ~3.7s 值得复用;ASR 协议本就 per-utterance。performance
AudioContext singleton不每轮 close+new。3-5 次重建后 mic 失活。reliability
Mic prewarm at startLessongetUserMedia + worklet 提前就绪。否则前几个字会丢。performance
ASR proxy PCM bufferupstream 握手期间 queue PCM。否则前 1-3 秒说话丢失。reliability
Worklet flushdisconnect 时刷剩余 buffer。否则尾字丢。reliability
800ms short-press reject前端直接拒,不进 thinking。豆包对 <1.5s 置信度低,立即反馈更友好。ux
5s ASR fallback5s timeout → awaiting。9s 太长,让重试不畅。ux
No barge-inspeaking 时空格忽略。清理 in-flight PCM 难做干净;后续再加。scope
Subtitle leads audio不做时间戳同步。画布跳得快 ~3s — 见 Known Issues。scope
Exact word match仅目标英文词精确 token 命中。保守优先;train 算对,tree 不算。reliability
ASR/TTS usage请求数 + 字符长度。provider 没暴露 token,先保证课后报告可观测。performance
Canvas 1:1图 75% (4:3 1024×768) · 文字 25%。幼儿教学:图为主角,文字为锚点。ux

8Protocol pitfalls

碰到豆包 ASR/TTS 报错时先翻这里,再翻 docs/DOUBAO Protocol/{asr,tts}.md。

Doubao V3 Bidirectional TTS

codec_flag_0x04must_know

Codec flag 锁 0x04(with event)。是否带 sessionId 由 event_id 决定(<100 不带、≥100 带、ConnectionStarted=50 带 connection_id)。不要根据"有没有 sessionId"切 flag,豆包会按 V1 sequence 协议解码报 sequence mismatch 伪错。

additions_is_jsonstringgotcha

req_params.additions 是 JSON 编码字符串,不是 object。

text_inside_req_paramsmust_know

TaskRequest 的 text 在 req_params.text,不在顶层。

long_conn_one_sessionmust_know

一个 WS 可多 session,同一时刻只能一个;session 间不重发 StartConnection。

Doubao V3 Streaming ASR

show_utterances_requiredmust_know

show_utterances: true 必传,否则无 utterances 字段,无法判 final。

result_type_fullmust_know

result_type: 'full',不是 'single'。single 增量,前面字会丢。

enable_ddc_falsemust_know

enable_ddc: false,儿童语料关掉语义顺滑。

hotwords_via_corpus_contextgotcha

targetWords 进 WS query → proxy 写 request.corpus.context = '{"hotwords":[...]}'。无 per-word weight,无 language_hint。

hotwords_limitsgotcha

2026-05-05 实测:hotwords 对 hour → Our.、1000 → 10 没救。字典化纠正已撤销(过拟合 + 让 LLM 看不见 raw ASR)。容错由 LLM attempt_assessment 承担。

client_finish_control_framemust_know

客户端不能直接 close WS;必须发 {type:'finish'},proxy 发 -seq 等 final。直接 close 卡 thinking。

proxy_must_buffer_pcmmust_know

client WS 几十 ms open,但 upstream 握手 1-3s。proxy 不 buffer 就丢前 1-3s。

Auth headers

ASR:X-Api-App-Key · X-Api-Access-Key · X-Api-Resource-Id · X-Api-Request-Id · X-Api-Sequence: -1

TTS:X-Api-App-Id · X-Api-Access-Key · X-Api-Resource-Id · X-Api-Connect-Id

注意 App-Key 与 App-Id 一字之差;X-Api-Sequence: -1 ASR 必传。
